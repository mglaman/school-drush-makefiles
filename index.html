So, what is a makefile?

For those who aren't familiar...
  • Built with Drush "make" command
  • Defines contributed and third party projects.
  • Turns into a working Drupal codebase.
  • Drupal distributions, like Panopoly, rely on makefiles.

How useful are they?

  • Define projects without bundling code in project.
  • Allows application of patches to fix bugs on projects.
  • Reduces version control clutter.

There is also specific patches included to fix bugs on projects before the release.

Imagine having patches modules in your code base and trying to catalog those changes.

Distributions often use many patches to provide a stable product before contributed projects can be updated.

Releases & patches.

Let's face it - Drupal codebases are complex.

  • Project updates are a file edit and rebuild.
  • Patches are managed in the makefile
  • Choose specific code revision, no more "latest dev"!

Complex: require multiple modules and libraries and patches to fix bugs.

Can specify revision, can pick when bugfix committed and not use -dev.

Facts

  1. Drupal is an open source community where most projects are maintained by volunteers
  2. You're going to need patches to fix bugs until the patches get comitted.
  3. Not a single project beside Core has a release schedule.

Makefiles

making life easier.

Lean repositories with only custom code providing easier management.

Anyone can read a makefile.

Everyone can know what projects are used at what version with what patches.
